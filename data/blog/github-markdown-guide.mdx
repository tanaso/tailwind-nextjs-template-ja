# はじめに

このブログテンプレートでは、`unified` やその他の `remark`、`rehype` パッケージを使用してMarkdownやMdxを解析します。`next-mdx-remote` を使用することで、`.mdx` や `.md` ファイルをWebpackを直接操作することなく柔軟に解析できます。

GitHub Flavored Markdown (GFM) を使用しており、`mdx-prism` によってコードブロックのシンタックスハイライトが可能です。以下にMarkdownのチートシートとデモを示します。

このMarkdownチートシートは、以下を参考にしています：
https://guides.github.com/features/mastering-markdown/

# Markdownとは？

Markdownは、ウェブ上でテキストをスタイルするための方法です。太字や斜体でのフォーマット、画像の追加、リストの作成などができます。Markdownの多くは、`#` や `*` などのアルファベット以外の文字を使ったシンプルな記述方法です。
